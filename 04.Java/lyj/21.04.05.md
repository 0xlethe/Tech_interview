## Reflection

- 정의된 클래스 / 메소드 / annotation 정보를 확인하고 참조하기
- 인스턴스를 생성하고, 필드값을 변경하며, 메소드를 실행할 수 있는 방법.

구체적인 클래스 타입을 알지 못해도 그 클래스의 메소드, 타입, 변수에 접근할 수 있도록 해준다.

### 장점

리플렉션을 사용하면, String 값을 통해 class를 얻어올 수 있고, 그 생성자를 얻어올 수 있다.

그 생성자를 통해 인스턴스를 생성할 수 있고, 생성된 인스턴스를 이용하여 메소드나 속성에 접근할 수 있다.



## Java 8 특징

- Stream API
- Lamda 표현식, **함수형 프로그래밍**이 처음으로 도입
- 나즈혼이라는 자비스크립트의 엔진 추가
- Method Reference, Default Method, Optional Class
- java.time 패키지 추가

### Stream API

데이터 요소인 배열이나 컬렉션 등의 데이터를 처리하기 위한 API

**Stream API 장점**

- 멀티 스레드를 활용하여 병렬로 연산 수행 가능

- 내부 반복으로 연산을 수행하기 때문에 코드 단순화 가능

  데이터 저장시 배열이나 컬렉션을 사용하는데 이에 접근하기 위해서는 반복문을 사용해야 했다. 또한 데이터마다 접근하는 방식도 달랐다.

  Stream API로 데이터를 추상화하여 공통된 방법으로 데이터를 읽고 쓰는 게 가능해졌다.

### Lambda

**람다표현식**

- 메소드를 하나의 식으로 표현
- 익명함수. 이름없이 실행할 수 있는 함수 표현식.

**Lambda 예시**

- 자바에서 Thread를 만들때 Runnable()객체를 할당하고 run 메소드를 실행하는데 이를 축약하여 ()로 나타낼 수 있다.

**Lambda 장점**

- 불필요한 코드 줄일 수 있다.
- 가독성을 높일 수 있다.

### 나즈혼 엔진

JAVA 8 이전에는 모질라의 리노 자바스크립트 엔진을 사용하였다. 이는 java의 최신 개선사항 등을 반영하지 못해 성능이나 메모리상으로 크게 향상된 나즈혼 엔진으로 업데이트 되었다.
